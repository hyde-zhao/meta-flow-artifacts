# CR-106 quant-lab remediation baseline and scope freeze

## 变更描述

Establish the P1 remediation baseline for quant-lab after adoption readiness: freeze remediation goals, non-goals, risk boundaries, affected surfaces, validation matrix, phase/CR plan, and entry criteria for later design without changing business code, credentials, runtime, production writes, trading, CR-033, or CR102-CR104 runtime gates.

## 不授权范围

- credentials / secret / account read
- runtime / SaaS / production write
- trading / live / publish
- CR-033 runtime trace follow-up activation

## 启动约束

- Formal CR IDs must use `CR-xxx`; `MF-xxx` is historical alias only.
- Business remediation starts only after CP0/context/human gate readiness is reviewed.

## P1 整改基线

### 范围目标

本 CR 只冻结 quant-lab 后续整改的范围、阶段、风险边界和验证入口，不实施业务代码整改。

| 维度 | P1 基线 |
|---|---|
| 主目标 | 将 quant-lab 从 adoption/precheck 状态推进到可设计的正式整改基线。 |
| 当前不纳入 | 真实 runtime、SaaS、trace upload、production write、交易、CR-033、CR102-CR104 runtime/NAS gate 收口。 |
| 允许动作 | 读取仓库源码/文档/过程文件；生成范围基线、阶段计划、CR 拆分建议、测试矩阵草案。 |
| 禁止动作 | 不访问 `.env`、token、API key、cookie、私钥、账号；不启动 QMT/MiniQMT/XtQuant/gateway；不改业务代码。 |
| 当前 adoption 状态 | workspace/state/ledger/quality/package identity 均已通过；默认 CR tracking 仍因多 active runtime CR 报错，用户声明 CR102-CR104 将在主线另行关闭。 |
