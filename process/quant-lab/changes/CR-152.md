# CR-152 Machine Learning Strategy E2E Framework Foundation

## 变更描述

启动 CR152 Machine Learning Strategy E2E Framework Foundation，先完成 CP0 受理、CP1 增量场景完备检查和 CP2 范围基线门。

CR152 的目标是基于现有 `ResearchDatasetSpec` / `BacktestRunSpec` / strategy admission 资产扩展 ML 策略专项契约，补齐 ML 策略从 PIT feature matrix、label policy、purged + embargo CV、training snapshot / model artifact metadata、prediction artifact 到 ML admission gate 的本地/static/fixture 框架基础。

CR152 当前不进入 CP3、Story 拆解、LLD 或实现；CP2 未获用户批准前，不得进行 Story 分解、LLD 设计批次或源码实现。

## 用户目标

用户要求按已确认路线 A 继续推进本地研究 / 策略框架补齐，下一步启动 CR152 Machine Learning Strategy E2E Framework 的 CP0 / CP2。当前优先级不是数据湖生产级闭环，也不是真实模型训练或 runtime 接入，而是先把 ML 策略 E2E 的基础契约和准入门控范围确认清楚。

## 范围

### In Scope

1. CP0 受理 CR152，记录 CR151 closed 基线和 STATE stale 残留刷新。
2. CP1 增量场景完备检查，基于既有 UC-59 / roadmap / remediation plan 确认本轮可进入 CP2。
3. CP2 范围基线门，确认 CR152 first wave 和不授权边界。
4. PIT feature matrix contract。
5. Label policy / leakage guard，且预留 `fixed_window` / `triple_barrier` / `meta_label` 扩展位；本轮不实现 triple-barrier 算法或真实 meta-label training。
6. Purged + embargo CV split policy and split audit contract。
7. Training snapshot / model artifact metadata。
8. Prediction artifact metadata。
9. ML admission gate，推荐新建 ML-specific admission gate，并通过 adapter 复用 CR151 四态 status 语义和 admission package linkage。
10. Fixture data contract：定义足以验证 purged + embargo CV 的最小 deterministic fixture schema、时间跨度、PIT 模拟方式和 fail-closed cases。

### Deferred / Later Wave Candidate

1. Feature importance report（MDI / MDA / SHAP 或项目选定等价解释工件）。
2. Sample uniqueness weighting。
3. Drift / retrain trigger。
4. Fractional differentiation。
5. Class imbalance / threshold tuning。
6. MinBTL / MinTRL gate。
7. 真实模型训练、模型注册、paper / live shadow。
8. CR153 Event-Driven Strategy E2E Framework。
9. CR154 Cross-Strategy Production Reliability Gates。
