# CR-109 quant-lab docs functional organization normalization

## 变更描述

Normalize docs/design, docs/features, and docs/quality around functional capability organization instead of CR-number organization, preserving historical CR evidence as legacy audit references and without business code remediation, credentials, runtime, production writes, trading, CR-033, or CR102-CR104 closure.

## 不授权范围

- credentials / secret / account read
- runtime / SaaS / production write
- trading / live / publish
- CR-033 runtime trace follow-up activation

## 启动约束

- Formal CR IDs must use `CR-xxx`; `MF-xxx` is historical alias only.
- Business remediation starts only after CP0/context/human gate readiness is reviewed.

## 变更原因

用户指出 CR108 关闭后 `docs/design`、`docs/features`、`docs/quality` 仍明显按 CR 编号组织；真实目标应按功能能力组织。该判断成立：CR108 只完成 current-truth 口径修订，未完成目录功能化。

## 目标

- 将 `docs/design`、`docs/features`、`docs/quality` 的 current-facing 入口按功能能力组织。
- 将 CR-scoped 历史材料迁入功能域下的 legacy / evidence 区，或保留但从 current index 中降级为 legacy audit reference。
- 保留历史审计语义，不批量改写 CR 文件正文。
- 不改业务代码、不读凭据、不运行 runtime。

## 初始目标功能域

| 功能域 | 目标目录 |
|---|---|
| market data lake | `docs/features/market-data-lake/` |
| factor research loop | `docs/features/factor-research-loop/` |
| execution semantics reference | `docs/features/execution-semantics-reference/` |
| qmt gateway readonly | `docs/features/qmt-gateway-readonly/` |
| qmt trading governance | `docs/features/qmt-trading-governance/` |
| runtime authorization safety | `docs/features/runtime-authorization-safety/` |
| qmt/miniqmt dual target framework | `docs/features/qmt-miniqmt-dual-target-framework/` |
| strategy research lifecycle | `docs/features/strategy-research-lifecycle/` |
| quant-lab migration dry-run | `docs/features/quant-lab-migration-dry-run/` |
| process / ledger / tooling hygiene | `docs/features/process-ledger-tooling-hygiene/` |

## 不完整项继承

CR108 已关闭但目录功能化未完成；CR109 承接该缺口。原计划的 CR tracking historical hygiene 顺延到下一张 CR。

## 执行结果

CR109 已按用户批准执行 docs 功能化目录迁移：

- `docs/features` 顶层已收敛为功能域目录；CR-scoped feature 目录迁入对应功能域的 `evidence/by-cr/`。
- `docs/design` 顶层已收敛为 current-facing 入口；CR-scoped design 文件迁入对应功能域的 `design/by-cr/`。
- `docs/quality` 顶层已收敛为 `README.md` 和当前 guardrail；历史质量证据迁入 `by-feature/<feature>/by-cr/<CR>/`。
- 迁移映射记录在 `process/checks/CR109-DOCS-FUNCTIONAL-MIGRATION-MAP.md`。
- 结构验证记录在 `process/checks/CP4-CR109-DOCS-FUNCTIONAL-ORG-VERIFICATION.md` 和 `process/checks/CP4-CR109-DOCS-FUNCTIONAL-ORG-VERIFICATION.result.json`。

本 CR 未执行业务代码整改、凭据读取、runtime、交易、production write、CR-033 或 CR102-CR104 关闭。
