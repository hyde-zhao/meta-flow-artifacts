## 变更描述

用户要求确认项目初始化相关技能是否已经包含 process 软链接建立逻辑。复核后发现总规则、README 和 CLI 已有外置 process 协议，但直接参与初始化的 `state-router` 与 `checkpoint-manager` 没有把“先建 artifact 目录和软链接，再初始化 STATE”写成硬前置条件；同时 `meta-flow workspace link` 对全新项目会因 `STATE.md` 尚未存在返回非零，不利于初始化流程。

本 CR 补强初始化入口：

- `state-router`：读写 `process/STATE.md` 前必须检查 process 路由；首次初始化必须先建立 `<artifact-root>/process/<project-name>` 和 `<project-root>/process` 软链接。
- `checkpoint-manager`：CP0 Entry Criteria / Checklist / Exit Criteria 增加 process 路由健康与软链接契约。
- `workspace link`：创建基础过程子目录，并把新项目的 `state_missing` 作为“下一步初始化 STATE”的成功后续，而不作为 link 命令失败。
