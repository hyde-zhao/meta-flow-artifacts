## 变更描述

CR-031 已补齐初始化入口和 CP 写入入口的 process 软链接检查。用户进一步确认：该规则不应只覆盖初始化，后续开发过程中凡是写入 `process/*` 的 skill 也必须先确认 route health。

本 CR 将一条条件式短规则同步到涉及 `process/` 路径且具备 `## 前置条件` 的功能 skill：

> 若本 skill 需要写入任何 `process/*` 文件，必须先确认 Host Orchestrator 已完成 process route health check；未确认时先交还 Host Orchestrator 执行 `meta-flow workspace check`，不得自行创建、修复或重建 `process`。

说明：

- `state-router` 与 `checkpoint-manager` 已在 CR-031 中具备更强、更细的专门规则，不重复插入。
- 对只读取 `process/*` 的 skill，该规则是条件式的，不额外要求写入动作。
- 对会写 CR、handoff、context capsule、Story、LLD、release context、discussion checkpoint 等过程产物的 skill，该规则形成后续开发过程的硬前置提醒。
