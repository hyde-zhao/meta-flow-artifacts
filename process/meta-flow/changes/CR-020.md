## 变更描述

新增 `meta-flow eval` CLI 子命令和 `meta_flow/evals/runner.py`，支持：

- `validate`：校验 `WORKFLOW-EVAL.yaml`、`PROMPT-BUNDLE.yaml`、`CASE-REGISTRY.yaml` 基础契约。
- `run`：执行确定性 graders 并写入 `run-summary.json` / `run-summary.md`。
- `suite-health`：汇总 `process/evals/runs/**/run-summary.json` 并生成 suite health。

## 五维度影响分析

| 维度 | 受影响对象 | 结论 | 处理动作 |
|---|---|---|---|
| 需求层 | 本地确定性验证能力 | true | 新增 CLI 子命令和 runner |
| 场景层 | schema、path、hash、case link、forbidden pattern | true | 首批 deterministic graders 覆盖样例 suite |
| 计划层 | CP7 eval evidence 输入 | true | runner 只生产 evidence，不替代 CP7 |
| 安全层 | no-network、no-credentials 默认路径 | true | 不接外部服务；forbidden pattern grader 检查敏感动作 |
| 交付层 | `meta_flow/evals/*`、CLI help | true | 可通过 `meta-flow eval ...` 运行 |
