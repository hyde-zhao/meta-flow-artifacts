# CR-036 approval-oriented CR and human gate use-case refresh

## 变更描述

Refresh meta-flow use cases for goal-oriented CR grouping, approval impact summaries, and risk-based human gate decision burden reduction.

本 CR 响应用户反馈：meta-flow 当前 CR 拆分有时过细，人工确认项也偏细，审批者难以理解某个 CR 对整体目标的作用、待决策内容的真实影响，以及 approve 后会改变什么。

## 目标影响摘要

| 字段 | 内容 |
|---|---|
| 目标包 | `GOAL-CR036-approval-ux` |
| 整体目标 | 让审批者在 CR 和人工门禁中先理解整体目标、影响和授权边界，再决定真正需要人工判断的事项。 |
| 影响的 use-case | UC-05 目标导向 CR 汇总；UC-06 带影响摘要的人工确认；UC-07 风险分级降低确认噪音；UC-08 新增需求入口分流 |
| 用户体验变化 | CR 汇总会解释该 CR 为什么存在、服务哪个目标、approve 后会发生什么；人工确认会先展示审批者摘要和决策分层，低风险实现细节默认由 agent 处理或审计记录。 |
| 决策负担 | reduced |

## 拆分理由

| 问题 | 结论 |
|---|---|
| 为什么这是一个 CR | 该变更聚合审批体验相关的 use-case、CR lifecycle、human-gate checker、ask-user 消息和交付文档，属于同一个审批体验目标包。 |
| 为什么不继续拆细 | 如果拆成模板、checker、CLI、文档多个 CR，审批者仍需要分别判断每个细项与整体目标的关系，会复现本 CR 要解决的问题。 |
| 为什么不合并到其他 CR | 没有仍在执行的父 CR 能完整覆盖审批体验目标；并入历史 CR 会掩盖用户反馈的独立目标和验收口径。 |
| 为什么不是普通 Story / Task | 该变更跨规则、CLI、模板、测试和文档，影响工作流治理协议，不只是单个实现任务。 |

## approve 后果

| 项目 | 内容 |
|---|---|
| approve effect | 接受完整方案后，meta-flow 将默认要求 CR / Human Gate 先输出审批者摘要和决策分层，并提供目标导向 brief 命令；低风险实现细节不再默认打扰用户。 |
| reject effect | 拒绝或要求调整会保留既有 CR-036 use-case 刷新，但不启用新的 human-gate 校验和目标包协议。 |
| approve 不授权 | 读取凭据或账户；真实运行外部服务；生产写入；publish / live / trading；自动关闭历史 CR；跳过 CP2 / CP5 / CP8 人工门禁。 |
