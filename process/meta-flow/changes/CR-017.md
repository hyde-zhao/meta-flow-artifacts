## 变更描述

用户要求优化 Meta Flow：取消 `meta-po` 作为主编排子 agent，改由主进程直接调度 `meta-pm`、`meta-se`、`meta-dev`、`meta-qa`、`meta-doc` 等功能子 agent，并要求同时适配 Codex 与 Claude Code。

本 CR 的核心边界：

- 删除的是 `meta-po` 子 agent 形态和安装入口，不删除编排、人工门禁、CR、Context Capsule、LLD clarification broker、状态推进和调度证据能力。
- 新默认模式为 `orchestration_model=host-orchestrated`，主进程为 `host-orchestrator`。
- Codex 继续使用 `.codex/agents/*.toml` 项目级功能子 agent；Claude Code 继续使用 `.claude/agents/*.md` 项目级功能 subagent。
- 功能 agent 不得接管 CP2 / CP3 / CP5 / CP8 正式人工门禁；正式门禁由主进程统一发起和回填。
