## 变更描述

用户反馈在其他项目使用 meta-flow 开发时，经常上报 `/home/hyde/projects/meta-flow/scripts/check_delivery_guardrails.py` 或 `scripts/check_delivery_guardrails.py` 找不到。

根因：`check_delivery_guardrails.py` 是 meta-flow 仓库自检脚本，不属于外部 production 项目的默认交付物；安装到其他项目的 rules / docs 不应把该脚本写成无条件提交前命令。

## 决策

| 决策点 | 结论 |
|---|---|
| guardrail 脚本归属 | `scripts/check_delivery_guardrails.py` 是 meta-flow 自身仓库 guardrail，不是外部项目默认文件 |
| 外部项目行为 | 若目标项目没有该脚本，不运行它；改按目标 README/docs 的测试、构建、安装 dry-run 或用户确认的验证命令执行 |
| 规则文案 | 所有 rules 必须写成条件执行：仅当当前仓库存在 `scripts/check_delivery_guardrails.py` 时运行 |
| 禁止项 | 不得在外部 production 项目中硬引用 `/home/hyde/projects/meta-flow/scripts/check_delivery_guardrails.py` |
