# MF-018 Implementation Summary

## Scope

- Implemented lightweight quality model and eval matrix policy templates.
- Added `meta-flow quality model-check` and `meta-flow quality eval-check`.
- Added `meta-flow doctor quality` and `meta-flow doctor workflow`.
- Added `meta-flow quality init` to initialize default quality governance policies for future projects.
- Kept workflow metrics derived from CP result JSON, event ledgers, and read-expansion ledger.
- Did not add a dashboard, agent ranking, portfolio reporting, or manually maintained workflow metrics truth source.
- Added delivery templates so installed context-manifest-builder skills can expose the default quality policy templates.

## Changed Objects

| Object | Path | Purpose |
|---|---|---|
| Policy template | `process/policies/QUALITY-MODEL.yaml` | Defines quality dimensions and derived-only metric sources. |
| Policy template | `process/policies/EVAL-MATRIX.yaml` | Maps quality dimensions to eval/check evidence. |
| Delivery template | `delivery/skills/context-manifest-builder/templates/QUALITY-MODEL-TEMPLATE.yaml` | Makes the quality model template available to installed skills. |

| Workspace compatibility | `meta_flow/workspace/routing.py` | Preserves symlink path metadata for `link_path`. |
| Tests | `tests/test_quality_governance.py` | Covers policy validation, CLI checks, and derived workflow metrics. |

## Notes For QA

- `doctor workflow` intentionally treats missing ledgers as warnings because metrics are derived from existing artifacts and should not force-create a metrics truth source.
- `process/policies/*` lives under the external `process` workspace symlink and is not listed by the source repository `git status`; delivery copies live under `delivery/skills/context-manifest-builder/templates/`.
- The full-test run exposed two compatibility failures outside MF-018; they were fixed minimally:
  - CLI workspace detection now recognizes `process/state/STATE.current.json`.
  - route metadata now records `link_path: "process"` without resolving the symlink target.
- Delivery route decision: include policy templates in `delivery/` and expose `meta-flow quality init` so future projects can initialize the policies without a manual metrics truth source.
